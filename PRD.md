# 나라장터 사전규격 공고 텔레그램 알림 봇 PRD

## 1. 프로젝트 개요
나라장터(G2B)의 '사전규격' 공고를 주기적으로 모니터링하여, 사용자가 설정한 키워드가 포함된 공고가 등록되면 즉시 텔레그램으로 알림을 보내주는 자동화 봇입니다. 
기존 '나라장터_입찰공고' 프로젝트의 구조(GitHub Actions 스케줄링 + 텔레그램 봇 API + 공공데이터포털 API 연동 및 상태 파일 저장)를 차용하여, 서버 구동 비용 없이 완전 무료로 작동하도록 설계합니다.

## 2. 주요 기능 및 특징
1. **사전규격 API 연동 및 데이터 수집**
   - 공공데이터포털 '나라장터 사전규격정보서비스' API 활용
   - 스케줄러를 통해 정기적으로 최근 등록/수정된 사전규격 공고 목록 조회
2. **맞춤형 필터링 엔진**
   - 사용자 정의 키워드 (OR, AND, 제외 단어 등 지원)를 기반으로 필터링
   - 특정 수요기관 필터링 기능 제공
3. **텔레그램 알림 발송**
   - 필터링된 신규 사전규격 공고를 지정된 텔레그램 방/채널로 전송
   - 메시지 내 핵심 정보 포함 (공고명, 수요기관, 배정예산, 의견등록마감일시, 상세 URL 등)
4. **중복 알림 방지 메커니즘**
   - 발송 완료된 사전규격등록번호를 상태 파일(`state.json`)에 기록
   - GitHub Actions 실행 완료 후 변경된 상태 파일을 Repository에 바로 Commit & Push 하여 중복 알림 차단
5. **명령어 기반 검색 (선택사항)**
   - `/search [키워드]`: 수동으로 텔레그램 내에서 즉석 사전규격 공고 검색 기능 지원
6. **서버리스 아키텍처**
   - 별도 서버 구축 없이 GitHub Actions의 Cron을 이용한 서버리스 파이프라인

## 3. 아키텍처 및 기술 스택
- **언어**: Python 3.11+
- **인프라**: GitHub Actions
- **외부 연동 API**:
  - 공공데이터포털 나라장터 사전규격 OpenAPI
  - Telegram Bot API
- **데이터 관리 방식**:
  - `state.json`: 알림 발송 이력 (JSON 텍스트 포맷)
  - `profiles.yaml`: 사용자별 검색 조건 관리
  - GitHub 본인 저장소 공간을 DB처럼 활용하여 상태 기록 유지

## 4. 상세 요구사항 명세
### 4.1. 프로필 관리 (`profiles.yaml`)
- 복수의 검색 프로필을 생성할 수 있어야 함.
- 각 프로필은 `키워드(or, and, exclude)`를 가질 수 있으며, 매칭 시 해당 프로필 단위로 파악되어 알림.

### 4.2. 중복 방지 로직 (`state_manager.py`)
- 이미 처리(발송)한 공고의 고유번호를 관리하여, 재실행 시 중복 알림이 전송되지 않도록 함.
- 파일의 크기가 무한정 커지지 않도록 오래된 기록은 주기적으로 정리(Truncate)하는 로직 필요.

### 4.3. GitHub 워크플로우
- 실행 주기: 매 30분 (예: `*/30 * * * *`) 권장
- 사용되는 GitHub Secrets 환경 변수:
  - `G2B_PREBID_API_KEY`: 공공데이터포털 사전규격 API 인증키
  - `TELEGRAM_BOT_TOKEN`: 텔레그램 봇 토큰
  - `TELEGRAM_CHAT_ID`: 알림을 수신할 채팅 ID
  - (선택) 로컬 테스트용 `.env` 제공.

## 5. 기존 프로젝트와의 차이점
기존 "나라장터_입찰공고" 봇과의 가장 큰 차이점은 **의존 API의 차이(입찰공고 vs 사전규격)** 입니다.
- 사전규격 공고 데이터는 `사전규격등록번호`, `품명`, `배정예산액`, `의견등록마감일시` 등의 고유 필드를 가지므로 데이터 모델(Pydantic) 및 출력 메시지 포맷의 변경이 필요합니다.
- 복잡한 입찰 모드 선택을 제거하고, "사전규격 단일 봇"으로 단순화 및 경량화합니다.
